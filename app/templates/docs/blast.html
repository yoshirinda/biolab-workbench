BLAST Search Documentation

Overview

BLAST (Basic Local Alignment Search Tool) is a sequence similarity search program. It compares query sequences against a database and finds regions of local similarity. BLAST is useful for identifying homologous sequences, functional annotation, and evolutionary studies.

Scientific Defaults by Scenario
Scenario Recommended Setup Why
Near-identical DNA program=blastn, task=megablast, strict E-value Fast and specific for high-identity nucleotide matches.
Short DNA query (<50 nt) program=blastn, task=blastn-short, smaller word_size Standard blastn can miss short high-quality local matches.
Protein homology program=blastp, keep SEG=yes, evaluate coverage + identity together Low-complexity masking reduces spurious high-scoring hits.
Cross-molecule search Use auto program selection (blastx/tblastn) Avoids query/database molecule mismatch and invalid interpretation.
Do not interpret hit identity alone. Always combine identity, query coverage, E-value, and biological context.
Parameter Documentation
Parameter Default Description When to Adjust
E-value 1e-5 Expectation value threshold. Represents the number of hits expected by chance. Lower values indicate more reliable matches. Scientific notation is supported (for example: 1e-5, 1e-10).
  • Strict screening: 1e-10 or lower
  • Relaxed search: 1e-3
  • Default suits most cases
Max Hits 500 Maximum number of aligned sequences to return.
  • Increase for comprehensive searches
  • Decrease for faster results
Min Identity % Disabled Post-filter on percent identity after parsing BLAST output.
  • Ortholog screening: start around 30-40% (proteins)
  • Close homologs: use 70%+
  • Never use identity without coverage constraints
Min Q-Covs % Disabled Post-filter on query coverage to remove short local alignments.
  • Functional inference: usually 50-70%+
  • Domain-level search: lower thresholds may be acceptable
  • Pair with identity and E-value
Program Auto-detect The BLAST program to use, based on query and database types.
  • blastn: DNA vs DNA
  • blastp: Protein vs Protein
  • blastx: Translated DNA vs Protein
  • tblastn: Protein vs Translated DNA
Task (blastn) Auto Fine-tunes nucleotide search behavior (megablast, dc-megablast, blastn-short). Use blastn-short for short oligos/primers and megablast for close homologs. In this workbench, blastn-short is auto-applied for short nucleotide queries (first query length ≤50) unless task is explicitly set.
Masking (DUST/SEG) Auto Low-complexity masking to reduce artifactual alignments. Disable only for explicit biological reasons and document that choice.
Word Size / Gap / Reward-Penalty Tool defaults Core alignment sensitivity and scoring controls. Change only when method-specific justification is clear (e.g., short query detection).
Common Use Cases

When searching for homologous genes in a specific organism:

  • Use E-value: 1e-5 for initial screening
  • Set Max Hits: 100 for manageable results
  • Use organism-specific database if available

When looking for distantly related sequences:

  • Use relaxed E-value: 1e-3 or 1e-2
  • Increase Max Hits to capture more candidates
  • Consider using blastp for better sensitivity with proteins

When reliable functional annotation is critical:

  • Use strict E-value: 1e-10 or lower
  • Look for hits with >50% identity and >70% coverage
  • Consider reciprocal best BLAST hits
Example Commands
Basic nucleotide BLAST:
blastn -query query.fasta -db database -out results.tsv -outfmt 6 -evalue 1e-5
Protein BLAST with detailed output:
blastp -query protein.fasta -db nr -out results.txt -outfmt 0 -evalue 1e-10 -max_target_seqs 100
Translated search (DNA query vs protein DB):
blastx -query dna.fasta -db swissprot -out results.tsv -outfmt 6 -evalue 1e-5
Output mode note: TSV and Detailed are tabular and suitable for filtering/extraction; Text is pairwise alignment view for manual inspection and is less suitable for structured downstream parsing.
Quick Reference
Recommended E-values:
  • 1e-50: Very high confidence
  • 1e-10: High confidence
  • 1e-5: Standard (default)
  • 1e-3: Sensitive search
Output Formats:
  • TSV: Best for parsing
  • Text: Human-readable alignments
  • Detailed: TSV with extra columns
